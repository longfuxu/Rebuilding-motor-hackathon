## 一、这个项目一句话
phi29 的 gp16 是一个由 **5 个相同零件(亚基)拼成的环形马达**,给病毒装 DNA。我们想把这 5 个零件**用共价键连成"可定点改造"的构件**——即能指定环上某一个位置放一个突变,而不是随机放。"能指定某一个位置"叫**可寻址(addressable)**。做出这样的构件,是为了将来用单分子光镊研究"环上各亚基怎么协同"。

---

## 二、我们做了几轮大迭代,每次为什么换方向

| 轮次(cycle) | 试了什么 | 撞了什么墙 → 学到的规矩 |
|---|---|---|
| **第 1 轮** | **环状置换**(circular permutation):把一个亚基从中间剪开、换地方重接,想变成单链 | 剪的位置**切进了一个结构域**(把一个折叠单元切碎),散架 → 规矩:**只能在结构域的边界剪** |
| **第 2 轮** | 在正确边界剪,先做 **2 个亚基**的融合 | 界面**时好时坏(随机)**;原因:gp16 的界面需要**两个邻居同时顶着**,2 个亚基只有一个邻居 → 规矩:**界面是"至少 3 个亚基"的性质,必须放进完整环里评估** |
| **第 3 轮** | 不剪,改成"**拴**":一个共价二聚体 + 3 个野生型,放进完整 5 元环里评估 | **成功**——界面像天然一样长回来了(这个构件叫 **B1**)→ 规矩:**拴、别剪;在环里打分** |
| **第 4 轮** | 用真正的进化信息把完整单链一节一节搭起来 | 完整单链在不同预测软件下**结论不一致**;查出是**中间连接段(linker)太长太松**,让链"穿错顺序" |
| **第 5 轮(当前)** | 缩短 linker → 两个软件下成了,但**第三个软件(AlphaFold3)在所有 linker 长度下都说"穿错"** | 结论:**直接用一根长链连不行** → 转向"**剪切拼接成 2–3 段共价件**"(见第七节) |

**主线**:剪 → 剪不行改拴 → 拴要在环里评 → 单链搭起来 → 单链 linker 太松会穿错 → 直接单链不行,改成 2–3 段共价件。

## 三、名词表(全称 + 解释)

**构件与方案:**
- **B1 / B2 / B3** — 第 3 轮里的三个方案。**B1** = 共价二聚体放进环里(**成功了,目前最稳的构件**);**B2** = 把设计规则画成"零件连接图"(一个方法工具,不是构件);**B3** = 用一段 RNA(pRNA)当支架固定亚基顺序的**另一条路**(备选)。
- **设计规矩 R1 / R2 / R3** — 上面表格里学到的三条(只在结构域边界剪 / 界面是 ≥3 亚基的性质、要在环里评 / 拴别剪)。
- **失败清单 F1–F6**(failure taxonomy) — 设计可能怎么死的清单(比如连接段堵住 DNA 通道、剪进催化位点),用来提前避坑。
- **L30 / L34 / L40** — 单链里**每段连接段(linker)的长度**(30 / 34 / 40 个氨基酸)。数字越大越松;越松越容易"穿错顺序"。

**生物学名词:**
- **精氨酸手指(arginine finger,在 gp16 里是第 146 位,常写作 R146)** — 一个亚基伸一根"手指"插进**邻居的 ATP 口袋**里。**这根手指有没有插对邻居,是判断界面成没成的核心标志。**
- **Walker A / Walker B** — ATP 结合(A)和水解(B)的两段保守序列。**催化位点 E119** 就在 Walker B 上。
- **E119Q** — 把第 119 位的谷氨酸(E)换成谷氨酰胺(Q),让这个亚基**催化功能死掉、但结构基本不变**;用来放在指定座位测"可寻址"(死掉一个指定位置,环还成不成、影响是否局部)。
- **N 端结构域 / C 端结构域(常写 NTD / CTD)** — 一个亚基的两半:**N 端**是 ATP 引擎(**精氨酸手指和 Walker 都在这**);**C 端**负责**和邻居拼成环**。

## 四、几个打分系统,怎么选(最重要的一条)

- **全局置信度(软件自带,叫 pTM、pLDDT)** — 软件对"整团东西折得像不像蛋白"的总体自信度。**问题:它分不清"正确的环"和"看起来漂亮但顺序穿错的环"**——事实上那个穿错的环,pTM 甚至更高、更"干净"。**所以绝不能用 pTM 判断环成没成,也绝不能拿它调参数。**
- **界面/几何指标(我们自己算的,叫 M1、M2、M4)** — 针对"功能"的判据:
  - **M2(核心)**:每个亚基的手指有没有正确搭到**"设计好的那个邻居"**的 ATP 口袋上(距离 < 8 埃)。5 个界面里 ≥4 个对上 = 成。
  - **M1**:5 个亚基的中心是不是排成一个**闭合的圆**,而且**顺序对不对**(排成圆但顺序穿错也算失败)。
  - **M4**:界面处的局部置信度。
  - **M3**:那个"特殊/开口位置"是不是固定的分析(叫 switch-position)。
- **一句话**:pTM 问"折得像不像蛋白";M2 问"这个环每个催化界面对没对上"。**我们只认 M1/M2/M4,不认 pTM。**

## 五、两个"阶梯(ladder)"别混

1. **融合阶梯(fusion ladder)** = 搭链过程:**2 个 → 3 个 → 4 个 → 5 个**一节一节往上搭,看在哪一节界面开始成/不成。**一节叫一个 rung**(2 个是低 rung,5 个是最高 rung)。
2. **保守性阶梯(conservation ladder)** = 改造手段从**最小改动到最激进**排成台阶:
   (0)天然环 →(1)拴天然序列 + linker →(2)调 linker 长度 →(3)**剪切拼接成 2–3 段 / 换结构域** →(4)用 AI 从头设计(RFdiffusion / LigandMPNN)。
   **原则:能用便宜的就别上贵的。** 我们现在停在第 (3) 步(剪切拼接);第 (4) 步生成式留作备选。

## 六、当前进度(2026-07-08)

- **最稳的构件仍是 B1**(共价二聚体 + 3 个野生型):Boltz-2、OpenFold3、Chai-1 都同意成环,而且**带催化死的 E119Q 在定点座位上照样成环**——可寻址性质具备。
- **直接用一根长链连的单链:废了。** 缩短 linker 让两个软件(Boltz-2、OpenFold3)成了,但**第三个软件 AlphaFold3 在所有长度(20/25/30/34/36/50)下都"穿错顺序"**;而且已确认**提交的序列没弄错**,所以这是真结果,不是操作失误。
- **带 ATP/镁 的天然环(AlphaFold3)——这个结果不可靠(已更正)**:实测环几何后发现,apo(不加 ATP)是**闭合平面环**(平面度 0.05 埃),AF3 加 ATP 后**还是闭合平面环**(平面度 0.01 埃),只是手指收紧到约 3.2 埃。它**没有**变成 ATP 结合应有的"开环/螺旋(lock-washer)"态(那个态就是实验结构 7JQQ)。原因:该任务开了 structure template(偏向闭合),而且这些预测器**只给一个静态结构、不建模 ATP 驱动的构象变化**。**结论**:配体态构象这个问题**超出结构预测器的能力**;7JQQ 本身就是实验上的 ATP 结合螺旋态。这写成一句"预测器的局限",而不是一个发现。

## 七、现在的策略:剪切拼接成 2–3 段共价件

**思路(你提出、参考 ClpX 那篇 central-coupler 论文)**:不再用一根 5 拷贝长链(会穿错),也不像 B1 留 3 个自由野生型;而是把五元环**拆成 2–3 段共价件,让它们互相拼装成环**。好处:每段短(二聚体约 684、三聚体约 1041 个氨基酸)→ 软件不容易穿错、物理上也更难乱拼,而且**每个位置都被共价固定,可寻址性比 B1 更强**。**先例**:单链 ClpX(把亚基共价连起来、定点放突变,记作 W/M)已被实验证明能装配、能工作,所以"共价多段连接"是站得住的;单链的"穿错"可能部分是软件对重复序列的预测毛病,而短片段能缓解。

**已经搭好、交给 Claude Science 折叠的构件**(`fold_inputs/cut_splice/`):
- **方案 A(2 段,3+2)**:三聚体(座位 1-2-3)+ 二聚体(座位 4-5)。
- **方案 B(3 段,2+2+1)**:二聚体 + 二聚体 + 单体。
判据:两个软件下整环 ≥4/5 个界面对上就算成;不成就调"剪切位点/连接段长度"。(注:更早的"天然 N-C 拴接"版 cut_splice 已作废,改用环状置换。)

## 八、我们的机理模型:环是"渐进式"开合(不是一次性)

这是我们对 gp16 环工作方式的模型,很重要——**别当成"两态(平面 vs 螺旋)"来分析**:
- **每结合一个 ATP,环就开一点点**;ATP 一个一个加上去(最多 5 个),环**逐步**从平面开向螺旋(lock-washer)。
- **ATP 水解成 ADP 时,环关回去一点**;等所有位点都变成 ADP/空,环回到**闭合平面**。
- 所以是**很多中间态**,由"有几个、哪几个亚基结着 ATP vs ADP/空"决定——一个随核苷酸依次点火而开合的循环。

**对我们已做分析的两个含义:**
1. **7JQQ 不是"完全打开的态",而是一个部分占据的中间态**(3 个亚基带 ATP 类似物 + 2 个空)。所以我做的 apo→7JQQ 重叠度,其实是 apo(0 个 ATP)→ 3 个 ATP 的中间态,是**部分开环**——正好符合"渐进式"这个模型。
2. **静态预测器(AF3/Boltz/OF3)对这套完全无能为力**(无论加不加 ATP 都给闭合环)。这套"占据数序列"最适合用**冷冻电镜在受控占据下解**(你们实验室),外加便宜的建模:**占据依赖的弹性网络 / 每亚基"核苷酸开关"模型**(把每个结着 ATP 的亚基往它的"开"局部构象推,松弛整环,看不同占据下的整体形状)。

**为什么这加强了我们的设计目标**:可寻址的单链/环状置换构件,就是用来在**定点**放一个催化死的亚基(E119Q),看它如何扰动**开环的顺序**——这正是直接检验"顺序点火 vs 齐步点火"的实验;也和我们之前"开口位置不固定、会移动"的发现一致。
